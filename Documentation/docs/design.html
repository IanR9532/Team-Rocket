1 Introduction

The Pokémon Encyclopedia will provide fans of Pokémon and Pokémon players all around the world with a comprehensive database of relevant information on Pokémon which will allow them to make educated decisions on how they would like to play the game.

1.1 Purpose

This design document will describe the architecture, functions, design, software interfaces and features of the Pokémon Encyclopedia.

1.2 Scope

The intended audience of this document will be the developers of the Pokémon Encyclopedia website. This document will be easy to use and learn from to help reduce the amount of time that is required to edit the website.

1.3 Definitions

2 Planned Implementation


2.1 Previously Planned Language

    Front-end: HTML, CSS

    Back-end: Python, SQL


    2.1.1 Updated Choice of Language

      Front-end: HTML, CSS, Bootstrap

      Back-end: Python, SQL, JavaScript, PHP

    Bootstrap: The Pokémon Encyclopedia is a web-based application and in addition to HTML and CSS, Bootstrap is also used for the front-end development to offer an interactive application. The design team decided to add Bootstrap as one of the front-end languages, since it provides a great grid system and it cooperates well with HTML and CSS. Bootstrap is used to extend functionality of the Pokémon Encyclopedia website, as it is a highly customizable language. If there is a demand for certain features, website developers can write code in Bootstrap and implement it on the web page with a third party add-on.

    JavaScript: In terms of the implementation of the Pokémon Encyclopedia app, using JavaScript would be helpful for real-time applications, as JavaScript is an event-based language with asynchronous nature. Hence, it is a good choice for the Pokémon Encyclopedia app, which requires constantly updated data.

    PHP: PHP is the top choice of language for the back-end implementation, since it is widely used to fetch data out of the database. This feature would provide users an ease of use experience, when they select their ideal Pokémon out of the comprehensive Pokémon Encyclopedia database. Another advantage of using PHP is it allows direct output to HTML, therefore whenever there is a modification or deletion to the database, the webpage is updated correspondingly and spontaneously. PHP is free to download and supports a wide range of databases, overall, it is the best tool for the Pokémon Encyclopedia application.

4 Interface Descriptions


4.1 Colour Guidelines

To maintain the Pokémon theme, we have created a standard color palette that will be used across the entire website. This was done to facilitate a sense of unity and harmony across the entire website while upholding the Pokémon theme. We first did some research on the official Pokémon color scheme. The main color scheme had very bold and intense colors, so we had to tweak them a little bit for them to be more appealing and less harsh on the eye when viewing on a screen.

5 Detailed Design

This section will describe the module design detail. Each interface and internal function which is implemented by a module or process will be described along with the steps followed to get from input to output, with the use of high level pseudocode. This pseudocode will be helpful in order to translate it into actual code during the implementation stage.

5.1 Module Detailed Design

In the Pokémon class, mostly all of the attributes are what describes the Pokémon. It includes: name, hp, attack, defense, special attack & defence, speed, evolve level, location, type and effectiveness. The methods will return the attributes of the Pokémon.

    5.1.1 Pokémon Class

    In the Pokémon class, most of the attributes describe the Pokémon. This includes: name, hp, attack, defense, special attack & defence, speed, evolve level, location, type and effectiveness. The methods will return the attributes of the Pokémon.

    5.1.2 Pokémon Class

    The Team class will be for the user's team of Pokémon . Attributes include owner, members, and number.

    Method Purpose
    getOwner()
    • Owner/User of the team
    • different owner for every team
    getMembers()
    • returns the list of Pokémon on the team
    setMembers()
    • sets list of Pokémon on team
    • different for every user
    5.1.3 User Class

    The user class will be for all of the user's information with attributes: username, user ID, password, saved teams and saved Pokémons.

    Method Purpose
    setUsername(username : String)
    • sets user's personal unique username into database
    • each user has a different username
    setPassword(password : String)
    • sets password for the user
    • connected to each user's username
    getSaved_teams()
    • returns list of user's teams
    • user can have multiple teams
    5.1.4 Buttons On The Back End

    On the login page, once you enter your username and password and click the sign in button you will get redirected to the homepage with their teams and Pokémon. If you don't have an account then you would hit the sign up button which would redirect you to the sign up interface via Ajax. If the user forgot their username or password, they can click the "Forgot Username / Password" button and it will lead them to enter their email and if they forgot their username it will send them an email with their username. If they forgot their password it will send them a validation code which can be entered in the next page and will allow them to reset their password.

    On the "Search Pokémon" page once they enter a Pokémon name to search and click the search button, it would launch a keyword search in the database and if its a valid Pokémon, it will display the Pokémon's information but if it's not then it will display a message saying the Pokémon name entered was not valid.

    On the "Search Pokémon" page, if they enter two Pokémon names and click the compare button, it will search for the two Pokémon's in the database and then display both of the Pokémon's attributes/information for the user to compare.

    5.1.5 Classes And The Database

    The Pokémon class has many attributes. The HP, Attack, Defence, Special Attack & Defence and Speed will be stored in the base_stats of the Pokémon in the database. The ID, Name and Evolve Level will be in the section "pokemon" of the database. The type of the Pokémon will be in a lookup section called "poke_type" of the database and there will also be a section called "types" which will list the Pokémon type with the corresponding type name. The location of where the Pokémon can be found will be in another lookup section called "poke_location" and it will also have another section called "location" which would list all the possible locations with the corresponding names.

6 Maintenance


6.1 Corrective

Pokémon Encyclopedia has a “get help” feature which allows users to report any issues or bugs with respect to the functionality of the app. Once the Pokémon Encyclopedia’s 24/7 chat line or email receives the requests from the users who see a problem and corrective maintenance is done by the implementation team. After corrective actions are taken, the implementation team will update the website by including the solutions and fixes.

6.2 Perfective

Due to the time constraint, the development of the app was simplified to the basic features needed in order to ensure the application is executable. Throughout the perfective maintenance, the implementation team will work closely on the reliability of the app and ensure that it is fully finished. Then, new functionalities and features are to be consistently updated once the application is reliable and complete.

In order to perfect the app, customer surveys will be done to collect feedback on users’ experience. Any suggested new capabilities or ideas will be considered by the implementation team, however the recommendations are ordered based on the popularity and the top suggestions are implemented first.

Some examples of features that is on the shortlist to be implemented in the application is listed below:

  • Add the ability to filter the Pokémon based on evaluation
  • Add different colour schemes for the theme of the app
  • Add chat functions to the app for users to talk to each other
  • Add a sharing viewing the Pokémon teammates have (of users who are in the same team)
  • Add a combat history record of Pokémon owned
