Entity Objects

There are two primary entity objects identified using noun-extraction. The project will store information about ‘first generation’ Pokemon and any teams created by the user

  1. Pokemon: the entity object that stores information about each Pokemon. It will allow Pokemon to be added to teams, to enable the user to easily see information about the Pokemon in one place, and to allow comparison between two Pokemon species. All attributes are set when entered into the database and immutable (unless changed at an administrative level inside the database).

    Attributes:

    • Name (string): the name of the Pokemon

    • HP (integer): the base health statistic*

    • Attack (integer): the base attack statistic*

    • Defense (integer): the base defense statistic*

    • Special Attack (integer): the base strength of the Pokemon’s special attack*

    • Special Defense (integer): the base strength of the Pokemon’s special defense move*

    • Speed (integer): the base speed statistic*

    • Evolve Level (integer): the level at which the Pokemon evolves into a different Pokemon (the value is NULL if it does not have an evolve level)

    • Location(s) (string): the location or locations where the Pokemon can be found in the game

    • Type (string): the Pokemon type statistic (water, fire, grass, rock, etc.)

    • Effectiveness (array): an array of all the strengths and weaknesses of a particular pokemon when fetched from the database.

    *Each Pokemon species has a base level of HP, Attack, Defense, Sp. Attack, Sp. Defense, and Speed regardless of the moves they have and level that they are.


  2. Team: the entity object that stores information about a Pokemon team (of max 6 Pokemon to match the in-game team size) that has been created. It enables the user to retrieve teams that they have created and saved earlier and then to make changes to them. All attributes in the team entity project are set when the user saves the team that has been created.

    Attributes:

    • Team Name (string): the name of the team, created by the user

    • Owner (User): the user who created the team

    • Members (list of Pokemon objects): the Pokemon on the team

    • Number (integer): the number of members out of 6


  3. User: the profile for the user that tracks their information. Is created by the user when they sign up to use the web application. Information is updated by them at their convenience (except their user ID which will not change).

    Attributes:

    • Username (string): the name of the user
      • Set during account creation

    • User ID (integer): a number to identify the user
      • Set during account creation

    • Password (string): the user’s password
      • Set during account creation

    • SavedTeams (a list of Team entity objects): the teams that the user has created and saved (up to three)
      • Set by the user during team creation or deletion, subject to change.

    • SavedPokemon (a list of Pokemon entity objects): Pokemon that the user has chosen to save
      • Set by the user after choosing to save or remove a Pokemon from their SavedPokemon list, subject to change.

Boundary Objects

Each boundary option has two sections to it. What details the information that the object must be input by the user and/or the information that will be presented to the user. How is a broad explanation on how the information will be presented and/or the action required from the user to obtain the information necessary.

  1. Log-in:

    2. What: The log-in menu will be used to record the user credentials and to identify the particular user. It will require two pieces of information from the user: the user ID and the password. If the credentials provided are not associated with a recorded user, an error message should pop up and ask for different credentials. Additionally, if the user forgets log-in password, the web page will direct the user to the help centre, where the user can reset the password through texts or emails.

    How: The user will type in the information. The error message and help centre will be a separate window.

  2. Registration:

    3. What: Presents the user with an option to register a new account. When a user attempts to create an account that already exists (based on the email being used), an error message will be displayed to indicate the registration failure. Another error message will be displayed if the information provided does not satisfy the requirements (e.g. username minimum/maximum lengths, etc.).

    How: The user will type in information. The error message(s) will pop up in a separate window.

  3. Home:

    4. What: It will present the user with four options that they can be directed to: the search option, saved Pokemon, saved teams, and comparisons.

    How: The object should have four buttons, corresponding to each of the above options, that the user can click.

  4. Search:

    5. What: The user must input the name of a Pokemon that they want to search. The Pokemon that matches the search will be presented to the user and the user will be prompted to view this Pokemon or search again. However, if the Pokemon is not found, the search will return none and prompt an error message - “There is no matching result”.

    How: The user will type in the information.

  5. Profile:

    6. What: The profile boundary object presents the user with all of the statistics in the Pokemon entity object (Name, HP, Attack, Defense, Special Attack, Special Defense, Speed, Evolve Level, Location(s), Type) and a photo of the Pokemon.

    How: A table of the statistics and a photo above it.

  6. Saved Pokemon:

    7. What: A list of the Pokemon that the user has chosen to save or favorite for future quick lookup.

    How: A list of the names of the Pokemon, sorted alphabetically.

  7. Comparison:

    8. What: The user must input which two Pokemon will be compared and then the user is presented with statistics (Name, HP, Attack, Defense, Special Attack, Special Defense, Speed, Evolve Level, Location(s), Type) from two Pokemon and their photos respectively. Should the user input two of the same Pokemon, they will receive a warning/error message stating that “The two Pokemon are the same,” and the program will only display one copy of the data.

    How: In a table, similar to the Profile object.

  8. Teams:

    9. What: Allows users to create teams and presents the user with a list of teams, up to three, that they have created. Users are not able to create more than three teams and if they try to, an error message will pop up to inform them that three teams at maximum is the rule.

    How: A list of the names of the teams, sorted alphabetically.

  9. Team:

    10. What: Presents the user with the information on a team, including a team name and Pokemon in the team. The user can input Pokemon that they would like to place on the team in an empty slot or to replace a team member. However, each team is only allowed to input up to six Pokemon, any member who attempts to input more than 6 pokemon will be prompted with an error message- “You’ve reached the maximum limit of Pokemon in a team, this Pokemon cannot be added.”

    How: The user will be able to click on slots and choose from a list. The slots will be presented to the user with six slots organized into two rows.

  10. Header:

    11. What: Presents the user with the title of the project (“Pokedex”) and the name of the current webpage. It enables the user to return to the Homepage.

    How: The user can press a button to return to the homepage.

Control Objects

The following control objects were determined based on verb extraction. The following descriptions detail what actions the control objects will take.

  1. Login Account: determine if the credentials entered by the user are correct (i.e. that they match to a saved user). If it is correct, it will allow the user entry to the website.

  2. Search: use the name entered by the user to find the Pokemon that matches it.

  3. Create Team: creates a team for the user by saving the name and Pokemon selected (as determined by the user).

  4. Retrieve Team: retrieves the saved team(s) from the database for viewing and modification by the user.

  5. Display Team: displays a team

  6. Save Pokemon: saves the selected Pokemon into the ‘Saved Pokemon’ list

  7. Retrieve Pokemon: retrieves a Pokemon from the database

  8. Display Pokemon: displays a Pokemon

  9. Compare: display two Pokemon for comparison
